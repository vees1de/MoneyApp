# Entity: certificate

## Как понял сущность
`Certificate` фиксирует факт успешного завершения обучения и служит источником для профиля сотрудника и отчетности.

## Что хранит
- Привязки: userId, courseId, requestId (опционально).
- Метаданные: number, issuedAt, issuer, validityPeriod.
- Файл и проверка: fileUrl/fileId, verified, verificationSource.
- Статус: uploaded/pending_verification/verified/rejected.

## Что бы я предпринял
- Разделить ручную и автоматическую верификацию сертификатов.
- Добавить проверку дублей и контроль формата файла.
- Определить политику хранения файлов и сроков доступа.
- Завести причину отклонения сертификата для прозрачной обратной связи.

## Вопросы для уточнения
- Кто валидирует сертификат: HR, тренер, автоматическая интеграция?
- Нужен ли срок действия сертификата и переаттестация?
- Обязателен ли шаблон данных сертификата для всех провайдеров?
