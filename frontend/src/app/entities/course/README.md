# Entity: course

## Как понял сущность
`Course` — центральная сущность контента обучения для внутренних и внешних программ.

## Что хранит
- Базовые данные: id, title, description, type (internal/external), status.
- Параметры: duration, format, provider, cost, tags.
- Управленческие признаки: mandatory, directionId, prerequisites.
- Связи: assessments, certificates, enrollments.

## Что бы я предпринял
- Ввести нормализованный справочник типов/форматов курсов.
- Разделить публичные и служебные поля (например, бюджетные ограничения).
- Добавить версионирование описания курса.
- Подготовить расширение под мультиязычный контент.

## Вопросы для уточнения
- Внешний курс всегда отдельная сущность или тоже `Course` с типом `external`?
- Нужны ли версии программы курса и архив старых версий?
- Нужен ли контроль уникальности по провайдеру + названию?
