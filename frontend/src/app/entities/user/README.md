# Entity: user

## Как понял сущность
`User` описывает сотрудника и его организационный контекст, через который завязаны назначения, заявки, согласования и отчетность.

## Что хранит
- Идентификаторы и профиль: id, ФИО, email, должность.
- Организация: departmentId, managerId.
- Доступ: role(s), permissions (если вынесены отдельно).
- Предпочтения: locale, timezone, theme.

## Что бы я предпринял
- Зафиксировать единый контракт `User`/`CurrentUser` для frontend и backend.
- Разделить PII и технические поля (для безопасного отображения).
- Добавить валидации и нормализацию контактов.
- Предусмотреть soft-deactivation (без потери истории).

## Вопросы для уточнения
- Возможны ли множественные роли у одного пользователя?
- Как обрабатываются внешние подрядчики/временные сотрудники?
- Нужна ли локализация ФИО и должности?
