# Feature: admin

## Как понял модуль
Модуль системного администрирования платформы: пользователи, роли, справочники, параметры и служебные настройки.

## Что делает модуль
- Управляет ролями, правами, маршрутами согласований и справочниками.
- Настраивает направления обучения, шаблоны уведомлений и лимиты бюджетов.
- Контролирует системные параметры и доступность интеграций.
- Дает инструменты служебного мониторинга и ручных операций поддержки.

## Что бы я предпринял
- Вынести в admin только то, что меняет глобальную конфигурацию системы.
- Реализовать экран управления справочниками с версионированием изменений.
- Добавить просмотр технических логов интеграций и очередей.
- Ограничить доступ строгой ролью `Administrator`.

## Вопросы для уточнения
- Кто имеет право управлять ролями: только super-admin?
- Нужен ли approval на изменение критичных системных настроек?
- Какие операции поддержки допустимы через UI, а какие только через backend tools?
