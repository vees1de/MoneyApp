# Feature: approvals

## Как понял модуль
Модуль оркестрирует маршруты согласования заявок: минимум Лид -> HR, с прозрачной историей решений.

## Что делает модуль
- Запускает маршрут согласования по этапам.
- Дает согласующим принять решение: approve/reject/request_changes.
- Хранит комментарии, SLA по срокам и audit trail.
- Возвращает итоговый статус в заявку и связанные процессы.

## Что бы я предпринял
- Ввести сущность `ApprovalStep` с порядком, ответственным и дедлайном.
- Настроить конфигурируемые маршруты по типу заявки/подразделению.
- Добавить автоматические напоминания и эскалации просроченных шагов.
- Реализовать экран очереди задач согласующего.

## Вопросы для уточнения
- Нужны ли параллельные шаги согласования или только последовательные?
- Кто может делегировать согласование на замещающего?
- Нужна ли финальная подпись администратором в отдельных случаях?
