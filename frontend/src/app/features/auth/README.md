# Feature: auth

## Как понял модуль
Модуль отвечает за вход пользователя в систему, управление сессией и проверку прав доступа по ролям (Администратор, HR/L&D, Руководитель, Сотрудник, Внутренний тренер).

## Что делает модуль
- Аутентификация: login/logout/refresh сессии.
- Авторизация: проверка ролей и разрешений для роутов и действий.
- Хранение данных текущего пользователя (профиль, роль, подразделение).
- Перенаправление после входа на роль-ориентированный ЛК.

## Что бы я предпринял
- Ввести `AuthFacade` + `AuthApi` + `AuthGuard` + `RoleGuard`.
- Добавить HTTP-interceptor для токена и обработки 401/403.
- Сконфигурировать единую модель `CurrentUser` для остальных фич.
- Сделать матрицу доступа по feature-модулям.

## Вопросы для уточнения
- Нужен ли SSO (AD/LDAP/OAuth2) или только локальная auth-схема?
- Где хранится источник ролей: в JWT или отдельным API?
- Нужна ли поддержка нескольких активных сессий пользователя?
