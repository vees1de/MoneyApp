# Feature: calendar-sync

## Как понял модуль
Модуль интеграции с Outlook: проверяет конфликты, создает события, обновляет расписание и напоминает о дедлайнах.

## Что делает модуль
- Проверяет доступность слотов сотрудника перед подтверждением обучения.
- Создает/обновляет календарные события при изменениях статуса.
- Проставляет ссылки на обучение в событии.
- Генерирует напоминания о дедлайнах и изменениях расписания.

## Что бы я предпринял
- Реализовать `CalendarProvider` с адаптером под Outlook API.
- Ввести модель `ScheduleConflict` и уровни критичности.
- Добавить идемпотентную синхронизацию событий (чтобы не плодить дубли).
- Вести журнал попыток синка и ошибок интеграции.

## Вопросы для уточнения
- Нужна ли двусторонняя синхронизация (из Outlook обратно в систему)?
- Что считать критическим конфликтом: overlap 100% или любой пересекающийся слот?
- Есть ли fallback, если Outlook временно недоступен?
