# Feature: notifications

## Как понял модуль
Централизованные уведомления для сотрудников, руководителей и HR о статусах заявок, дедлайнах и изменениях расписания.

## Что делает модуль
- Формирует уведомления по событиям из модулей заявок, согласований и прогресса.
- Отправляет через каналы (in-app, email; push опционально).
- Поддерживает шаблоны сообщений и планировщик отправки.
- Хранит историю и статус доставки.

## Что бы я предпринял
- Сформировать каталог событий и шаблонов уведомлений.
- Ввести приоритет сообщений (критично/обычно/информационно).
- Добавить user-preferences по каналам и quiet hours.
- Реализовать retry + дедупликацию отправок.

## Вопросы для уточнения
- Какие каналы обязательны для MVP: только in-app+email?
- Нужны ли юридически значимые уведомления с подтверждением прочтения?
- Какие SLA по времени доставки критичных уведомлений?
